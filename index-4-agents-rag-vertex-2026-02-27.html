Entendendo a "Distância" na IA

Ao configurar pesquisa vetorial (ex.: Vertex AI, Pinecone, Milvus), você escolhe como o sistema compara vetores. As opções — Euclidiana, Cosseno, Produto Escalar — são métricas de distância ou medidas de similaridade: fórmulas que, dados dois vetores, retornam um número. Não são tipo de vetor nem um único algoritmo de busca; são a regra usada para ranquear vizinhos. Este material serve como referência: fórmulas, exemplos em Python, quando usar e fontes oficiais.

📐 O que é um vetor?

É a representação numérica de um item (filme, texto, imagem). Uma lista de números que descreve o item em várias dimensões. Ex.: [Ação=9, Comédia=2]. Os vetores são gerados por um modelo de embeddings; você não escolhe "tipo de vetor" na tela de distância.

📏 O que é métrica (Euclidiana, Cosseno, etc.)?

É a fórmula matemática usada para comparar dois vetores e obter um número. Define se "parecidos" = distância pequena ou similaridade grande. Não é um algoritmo de busca em si: é a regra que o motor usa para ranquear os vizinhos mais próximos.

Onde isso aparece: Vertex AI Vector Search (distanceMeasureType), scikit-learn (metric= em k-NN e pairwise_distances), bancos vetoriais (Pinecone, Weaviate, etc.).

🔍 O Cenário:

Imagine que a IA precisa saber se o filme "Duro de Matar" é parecido com "Vingadores". Ela transforma cada filme em um vetor (números) e usa uma métrica escolhida por você para medir quão próximos são.

Métrica de distância

1. Distância de Manhattan (L1)

O que é: Métrica de distância (norma L1) que soma o valor absoluto das diferenças em cada dimensão — também chamada city block. É a função usada para comparar dois vetores; não é tipo de vetor nem algoritmo de busca.

Analogia do taxista. Em Nova York você não atravessa prédios: anda por quarteirões. A “distância” é a soma dos deslocamentos em cada eixo.

Fórmula

d(a, b) = |a₁ − b₁| + |a₂ − b₂| + … + |aₙ − bₙ| = Σᵢ |aᵢ − bᵢ|

Exemplo em Python 

import numpy as np
from sklearn.metrics.pairwise import manhattan_distances
a = np.array([[9, 2]])
b = np.array([[2, 8]])
print(manhattan_distances(a, b)[0, 0])  # 13.0

Quando usar

Exemplo prático — rota em quarteirões: Você está no ponto (2, 3) e o delivery no (7, 8). Não pode cortar caminho: soma dos blocos na horizontal (5) + na vertical (5) = 10 blocos. É exatamente a distância de Manhattan. Em jogos de tabuleiro (x, y) ou logística em ruas em grade, L1 modela o deslocamento real.
  • • Grade/ruas, tabuleiros, eixos independentes.
  • • k-NN: KNeighborsClassifier(metric='manhattan') — menos sensível a outliers.
  • • Dados esparsos (scikit-learn suporta).
  • • Evitar para embeddings de texto (preferir cosseno/produto escalar normalizado).

Fontes: scikit-learn, Vertex AI

Métrica de distância

2. Distância Euclidiana (L2)

O que é: Métrica de distância da linha reta entre dois pontos (Teorema de Pitágoras, norma L2). Menor valor = vetores mais próximos. Sensível à escala das features — normalizar ou padronizar antes de usar.

Analogia do pássaro. O pássaro voa em linha reta; a distância é o comprimento desse segmento.

Fórmula

d(a, b) = √[(a₁−b₁)² + (a₂−b₂)² + … + (aₙ−bₙ)²] = √(Σᵢ (aᵢ − bᵢ)²)

Exemplo em Python 

import numpy as np
from sklearn.metrics.pairwise import euclidean_distances

a = np.array([[9, 2]])
b = np.array([[2, 8]])
print(euclidean_distances(a, b)[0, 0])  # ~9.22

# k-NN com L2 (padrão): NearestNeighbors(metric='euclidean')

Quando usar

Exemplo prático — “restaurantes perto de mim”: Dois pontos no mapa: você em (lat, lon) e cada restaurante em (lat, lon). A distância em linha reta (como o pássaro voa) é a Euclidiana. Apps de delivery e mapas usam essa ideia para “mais próximo”. K-Means para agrupar lojas por localização também usa L2: cada loja fica no grupo do centroide mais próximo em linha reta.
  • • k-NN, clustering (K-Means), detecção de anomalias — padrão em muitos algoritmos.
  • • Dados numéricos em escala similar; fazer StandardScaler/Normalizer quando as escalas forem diferentes.
  • • Cuidado com texto/embeddings: frase curta vs longa com mesmo significado podem ficar “distantes”; para semântica costuma-se usar cosseno ou produto escalar com vetores normalizados.

Fontes: scikit-learn, Vertex AI (SQUARED_L2_DISTANCE)

Métrica de distância

3. Distância do Cosseno

O que é: Métrica baseada no ângulo entre os vetores: similaridade de cosseno = cos(θ) ∈ [−1, 1]; distância do cosseno = 1 − cos(θ) (0 = mesma direção, 2 = opostos). Não depende da magnitude — só da direção. Ideal para texto e embeddings semânticos.

Analogia da direção. Como apontar para uma estrela: o braço pode ser curto ou longo; o que importa é a direção.

Fórmula

similaridade = (a·b) / (‖a‖ ‖b‖) = cos(θ)
distância = 1 − similaridade

Exemplo em Python 

from sklearn.metrics.pairwise import cosine_similarity, cosine_distances
import numpy as np

a = np.array([[9, 2]])
b = np.array([[2, 8]])
sim = cosine_similarity(a, b)[0, 0]
dist = cosine_distances(a, b)[0, 0]  # 1 - sim
print(f"Similaridade: {sim:.4f}, Distância: {dist:.4f}")

Quando usar

  • • Busca semântica, RAG, NLP: “Amo cães” e “Adoro muito os cachorros” — mesma direção, tamanhos diferentes.
  • • Embeddings de texto (BERT, sentence-transformers): comparação por significado, não por tamanho do texto.
  • • Recomendação por conteúdo quando a “força” do vetor não importa.
Performance: Exige norma e divisão em tempo de consulta; para milhões de vetores o Vertex AI recomenda DOT_PRODUCT_DISTANCE + UNIT_L2_NORM em vez de COSINE_DISTANCE (mesmo ranking, mais rápido). Ver seção Otimização.

Fontes: scikit-learn, Vertex AI

Medida de similaridade

4. Produto Escalar

O que é: Medida de similaridade (não distância): a·b = a₁b₁ + a₂b₂ + … + aₙbₙ. Quanto maior, mais similares. Considera direção e magnitude. Com vetores normalizados (‖a‖=‖b‖=1), produto escalar = similaridade de cosseno — mesmo ranking, cálculo mais barato.

Analogia do “high-five”. Para o “bater” ser forte, precisa da mesma direção e de força (magnitude).

Fórmula

a·b = Σᵢ aᵢbᵢ      Se ‖a‖=‖b‖=1 então a·b = cos(θ)

Exemplo em Python 

import numpy as np

a = np.array([9, 2])
b = np.array([2, 8])
dot = np.dot(a, b)
print(dot)  # 34

# Com normalização L2 = similaridade de cosseno:
from numpy.linalg import norm
a_n = a / norm(a)
b_n = b / norm(b)
print(np.dot(a_n, b_n))  # igual a cosine_similarity([a],[b])

Quando usar

Exemplo prático — busca semântica em produção: Você quer o mesmo efeito do cosseno (ex.: perguntas como “Amo cães” retornando respostas sobre cachorros), mas com performance melhor. No Vertex AI (e em outros serviços) você configura DOT_PRODUCT_DISTANCE + UNIT_L2_NORM: os vetores são normalizados e o produto escalar entre eles dá o mesmo ranking que a similaridade de cosseno, só que calculado mais rápido. É a recomendação oficial do Google para esse cenário.
  • • Busca vetorial em escala (Vertex AI, Milvus, etc.): com UNIT_L2_NORM, DOT_PRODUCT_DISTANCE = mesmo ranking do cosseno, mais rápido.
  • • Quando a magnitude importa: e.g. intensidade de preferência em recomendações (vetores não normalizados).
  • • Implementações otimizadas (SIMD, GPU): só multiplicações e somas, sem raiz nem divisão.

Fontes: Vertex AI — DOT_PRODUCT_DISTANCE, scikit-learn cosine_similarity

AVISO DO GOOGLE Por que "Não Recomendado"?

Se o Cosseno é tão bom para textos, por que o Vertex AI diz que é [altamente não recomendado]?

A resposta é Performance. Calcular o cosseno para milhões de vetores é lento. Mas existe um Truque de Ouro matemático:

✨ O Segredo da Normalização

Se todos os vetores tiverem norma 1 (normalização L2), o Produto Escalar é igual à Similaridade de Cosseno. O ranking de resultados é idêntico; a diferença é de performance.

Documentação Vertex AI: “We strongly suggest using DOT_PRODUCT_DISTANCE + UNIT_L2_NORM instead of the COSINE distance. Our algorithms have been more optimized for the DOT_PRODUCT distance, and when combined with UNIT_L2_NORM, it offers the same ranking and mathematical equivalence as the COSINE distance.”

Fonte: Vertex AI — DistanceMeasureType

Na prática: Configure o índice com featureNormType: UNIT_L2_NORM e distanceMeasureType: DOT_PRODUCT_DISTANCE. Se seus embeddings já forem normalizados pelo modelo, o Vertex aplica a normalização de qualquer forma quando o tipo for UNIT_L2_NORM. Você obtém o mesmo comportamento do cosseno com melhor desempenho.

Normalização em Python 

from numpy.linalg import norm
import numpy as np
def normalize_l2(X):
    norms = norm(X, axis=1, keepdims=True)
    norms[norms == 0] = 1
    return X / norms
# Exemplo: embeddings (n_vectors, dim)
emb = np.array([[9, 2], [2, 8], [5, 5]])
emb_norm = normalize_l2(emb)
